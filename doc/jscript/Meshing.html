eliminateInternalVertices

bool eliminateInternalVertices()

Description:
*Remove Internal Vertices* Eliminate points not required to represent the model. This option can be used to get better mesh around holes with buckling stiffeners.

eliminateNonConceptualEdges

bool eliminateNonConceptualEdges()

Description:
*Eliminate non-conceptual short edges* Mesh edges shorter than specified value will be collapsed even if they are not caused by short model edges. This is done as a post-meshing operations, as opposed to the elimination of short model edges. Note that collapsing short edges may result in new edges which are still shorter than the threshold value. For that reason, the user may specify a number of times this operation will be performed. For performance reasons, this number should be as low as possible (preferably 1-3).

meshDensityRounded

bool meshDensityRounded()

Description:
*Round off Mesh Density* The default rule is that elements are shorter than or equal to mesh density. Thus, the number of elements along an edge is obtained by rounding up (edgeLength/meshDensity). Turning on the "Round off Mesh Density" option allows for slightly larger elements. This is achieved by instead rounding up (edgeLength/meshDensity - 0.5).

useMinMaxDensityRatio

bool useMinMaxDensityRatio()

Description:
*Adjust edge density using ratio* Density on edges with no enforced density will be calculated from the minimum and maximum density specified on the other edges of a face. The density will be set to maximum of minimum face density or ratio multiplied by maximum density. I.e. if ratio is 0, minimum density is used and if ratio is 1.0 maximum density will be used. Values outside the range 0-1 is not accepted. This feature is especially useful for areas of the model where the user does not want to specify a density.

useThickOverrideThinShellElements

bool useThickOverrideThinShellElements()

Description:
*First Order Thick Shell Elements* In the analysis, first order thin shell elements will be overridden by first order thick shell element in Sestra. The 3 & 4 node thick shell elements are suited for models where the element thickness is significant relative to the element mesh size and where out of plane shear forces are significant, but are not necessarily better in all other situations. In the Sestra analysis run, the option THCK is set to 1. on ELOP card in the Sestra input file. Sestra 10.11 or later is required.

useUniformizedFaceParameterization

bool useUniformizedFaceParameterization()

Description:
*Uniformized Face Parameterization* For certain surfaces, the default parameterization provided by Acis is unsatisfactory for the low-level face mesher. To improve the mesh of surfaces with complex geometry, try enabling this option. Note, however, that this will significantly reduce meshing performance.

writeLoadsSeparate

bool writeLoadsSeparate()

Description:
*Write loads separate* Should normally be enabled, since memory usage is reduced. Disable the option during problem solving if some loads appear to be missing on FEM model, and check if this has any positive effect.
