includeLoadsOnMesh

bool includeLoadsOnMesh()

Description:
*Enables/Disables load generation on mesh.* When checked, loads will be applied to the mesh after mesh generation. When unchecked, loads will not be applied until an analysis is run. Uncheck to speed up meshing process.

linearSpringsAtPileHeadSubset

Set linearSpringsAtPileHeadSubset()

Description:
*Set containing support points (for ESL analysis only).* The set should contain all support points with springs that are located at pile head locations. If no set is specified, then it is assumed that all support points with springs in the model are located at pile heads.

multithreadedLoadApplier

bool multithreadedLoadApplier()

Description:
*Parallel load generation on mesh.* When checked, loads will be computed in parallel, which may increase performance on models with many load cases and elements. When unchecked, the loads will be computed sequentially. Uncheck if instabilities are encountered during load applications.

multithreadedMesher

bool multithreadedMesher()

Description:
*Parallel mesh generation.* When checked, the mesh generation will be executed in parallel, which may increase performance on models with many elements. When unchecked, the mesh is generated sequentially. Uncheck if instabilities are encountered during mesh generation.

regenerateMeshOption

RegenerateMeshOption regenerateMeshOption()

Description:
*Regenerate mesh information* The first option is identical to behaviour in previous GeniE releases. The limitations for the two next options are listed in the user manual. *Always Regenerate Mesh* Perform a full regeneration of loads and mesh, every time the meshing is executed. *Conditional Regenerate Mesh* The program will decide if regeneration or partial regeneration of loads and mesh is required, or if an update of loads and properties is sufficient.

smartLoadCombinations

bool smartLoadCombinations()

Description:
*Enables/Disables use of smart load combinations.* When checked, scaled results of combinations are created by GeniE when the analysis is completed. When unchecked, these loads are written explicitly to file prior to analysis. Prior to exporting the model to Usfos the analysis must be executed with this option unchecked.

writeComputedLoadCasesIntoCombinationBSELL

bool writeComputedLoadCasesIntoCombinationBSELL()

Description:
*Enables/Disables writing computed basic load cases into load combination BSELL cards.* When checked, the load combinations from GeniE are represented as BSELL includes computed basic load cases. This option must be unchecked when performing ESL for ULS Pile-Soil analysis using Sestra 10.

writeFEMFile

bool writeFEMFile()

Description:
*Enables/Disables writing of FEM file.* When checked, a FEM file describing the mesh will be written after meshing. This file is necessary for running analyses, but not for investigating the mesh. Uncheck to increase performance during design iterations. GeniE will postpone FEM file generation until needed.

writeLoadCombinationsOnFirstLevelAsBSELL

bool writeLoadCombinationsOnFirstLevelAsBSELL()

Description:
*Enables/Disables utilization of BSELL cards.* When checked, the load combinations from GeniE are represented as BSELL cards on the first level super-element. This option is recommended to be used when performing Pile-Soil analysis using Sestra 10. Prior to exporting the model to Usfos the analysis must be executed with this option unchecked. Note that BSELL cards on a first level super-element can not be interpreted and handled by PRESEL.
