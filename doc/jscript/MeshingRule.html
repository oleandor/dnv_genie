Command: MeshingRule

Description:
Global settings for mesh creation
Example of use:
Global settings for mesh creation
The MeshRules are parameters that guide the meshing. There are a number of settings, some with additional options.
Each setting can be activated or deactivated with the activate method.
The value used can be set of changed with the setLimit method.
Some of the settings require an additional option. The option is the action the mesher shall use.
Each option can be activated or deactivated separately.
E.g. it is possible simultaneously to tell the mesher to fail if the max angle of the element exceeds 179 degrees and to split the element if the max angle exceeds 165 degrees.
These settings applies to the whole model, and thus the values are scaled to be size independent.
Although there are default values and some suggested values below, the appropriate settings should be determined by the project, e.g. through parameter studies.
Explanation of the settings:
mpMaxAngle -> Maximum included angle between edges of a quad element.

mpMinAngle -> Minimum included angle between edges of a quad element.

mpMaxChord -> Limits the maximum free chord within one element for a curved geometry edge.
Increases the mesh density if the deviation between the straight element edge and the actual curvature is to large.
The tolerance is the distance from the midpoint of the element edge to the chord of the actual curvature.

mpMinEdge -> Makes sure that small details in the geometry model does not lead to tiny element edges.
The value is relative to the mesh density for the part.

mpMinEdgeLength -> Makes sure that small details in the geometry model does not lead to tiny element edges.
The value is the minimum length of edges.

mpMaxRelativeJacobi -> Maximum absolute Jacobi determinant of a node divided by the minimum
The Jacobi determinant meassures the quality of the quad element. If the relative value is 1, the element is a perfect square.
If the relative value is large the element may be seriously misshaped, and unacceptable to the analysis program.
The Jacobi determinant is calculated for each node. The maximum and minimum absolute values are found, and the relative size is used.

mpMinNormalizedJacobi -> Minimum normalized Jacobi determinant of the nodes of quad elements.
The Jacobi determinant shall be larger than 0 for all nodes of an acceptable element. Small positive values indicates bad shape.
The determinant is normalized to make the minimum value equal to 1 for the perfect square elements. Minimum normalized Jacobi determinant of the nodes of quad elements.
Explanation of the options
mpFail -> If the value is exceeded, the meshing will not produce any mesh for the face.

mpSplit -> If the value is exceeded, the meshing will produce two triangular elements by splitting the quad.

mpMeshDensityRounded

Preference mpMeshDensityRounded()

Description:
Global preference for interpretation of mesh densitySetting this value changes (globally) the interpretation of the mesh density set on each part. The default value of the parameter is "false".

Mesh density is among other things used to calculate the number of elements along any given edge, i.e. number_of_elements = edge_length/mesh_density

Obviously, the division is a floating point number that must be rounded up or down to yield an integer value (an edge cannot have 3.14 number of elements for example).

  • Setting mpMeshDensityRounded to true will imply using normal rounding rules (e.g. 3.14 is rounded to 3 while 3.54 is rounded to 4).
  • Setting mpMeshDensityRounded to false will always result in rounding up (i.e. both 3.14 and 3.54 results in 4 elements along the edge).
