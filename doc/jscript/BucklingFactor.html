Command: BucklingFactor

Description:
Beam buckling factor property. ky or effectiveLengthFactorY and kz or effectiveLengthFactorZ are the buckling factors,
bucklingLengthY and bucklingLengthZ are the buckling lengths and momentAmplificationY and momentAmplificationZ are the moment amplifications
in the respective directions.
Example of use:
Define local buckling factors for individual beams
Bm21.buckling = BucklingFactor(0.9, 0.8);
This command is scripted. Typically when specifying buckling factor of 0.9 and 0.8
for buckling around local y and z-axis. These factors can be referred to when defining a
capacity model. These factors are also transferred to Framework if you want to do code checking
based on older code checking standards. See User Manual Volume 4 for more details.
Define local buckling data for individual beams
Bm22.buckling = BucklingFactor(bucklingLengthOptionBeamLength, bucklingLengthOptionBeamLength, 0.9, 0.8, 0.85, 0.85);
This is an example of a command for specifying default buckling lengths,
effective length factor of 0.9 and 0.8 and moment amplifications of 0.85
for buckling around local y and z-axis. These factors can be referred to when defining a
capacity model. These factors are also transferred to Framework if you want to do code checking
based on older code checking standards. See User Manual Volume 4 for more details.
Define extended local buckling data for individual beams
Bm22.buckling = BucklingData(bucklingLengthOptionBeamLength, bucklingLengthOptionBeamLength, 0.9, 0.8, 0.85, 0.85, stiffenerSpacingOptionNone, stiffenerSpacingOptionNone, 0.97, unbracedLengthOptionNone, unbracedLengthOptionNone);
This is an example of a command for specifying default buckling lengths,
effective length factor of 0.9 and 0.8, moment amplifications of 0.85, default stiffener spacing,
bending coefficient of 0.97 and default unbraced length values for buckling around local y and z-axis. These factors can be referred to when defining a
capacity model. These factors are also transferred to Framework if you want to do code checking
based on older code checking standards. See User Manual Volume 4 for more details.
