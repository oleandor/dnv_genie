Command: Joint

Description:
Represent a tubular joint
Example of use:
Insert a joint by specifying its coordinates manually
Jt1 = Joint(Point(4 m,11.5 m,10 m));
This command is scripted. Typically, when defining a joint Jt1 at a beam intersection at (4m, 11.5m, 10m)
Insert a joint by snapping to 1 point. Same conditions as above apply
Jt1 = Joint(Point(4 m,11.5 m,10 m));
This command is scripted. Typically, when defining a joint Jt1 at a beam intersection at (4m, 11.5m, 10m)
Insert joints according to the joint creating rules by selecting a beam or beams.
Jt2 = Joint(Point(0 m,28.5 m,10 m));
Jt3 = Joint(Point(4 m,28.5 m,10 m));
Jt4 = Joint(Point(12 m,28.5 m,10 m));
Jt5 = Joint(Point(18 m,28.5 m,10 m));
Jt6 = Joint(Point(24 m,28.5 m,10 m));
Jt7 = Joint(Point(27 m,28.5 m,10 m));
This command is scripted. Typically, when referring to a beam with 6 intersections a total of 6 joints are created.
In this case the rules include that joints shall be created at each beam intersection

autoCone

  autoCone(Beam beam, int iseg)

Description:
Add a cone to the specified segmentThis command will add a cone to the neightbour segment of this segment if the specified segment has a reinforcement property. If this segment is used to represent the reinfocement (typically can) on both sides of a joint, this command will add two cone sections. The cone section used is specified in the JointDesign rule

flushBraces

  flushBraces()

Description:
Flush all braces to the surface of the chordIt is possible to turn on automatic flushing of braces in the JointDesign rule. If you want all your braces to be flushed, this is the recommended practice. Limitations:
In the cases where braces intersects the chords, for example X-joints, the braces will not be flushed.
If the braces are flushed after cans and stubs are generated, This command will not recalculate the length of the cans and stubs.
