Command: PileSoilAnalysis

Description:
Define a Wave Load Analysis
Example of use:
Execute and control input data for a pile soil analysis
Analysis1 = Analysis(true);
Analysis1.add(MeshActivity());
Analysis1.add(WaveLoadActivity(Condition1));
Analysis2.add(PileSoilAnalysis(Condition1));
Analysis1.add(LinearAnalysis());
Analysis1.add(LoadResultsActivity());
Analysis1.execute();
This command is scripted provided that you tick off for "Journal activity executions".
It is required to define a wave load condition that can be referred to when defining the wave load activity
(Insert|Environment|Deterministic Time Condition). For more details, see User Manual Volume 2 or 6.
Nonlinear Calculation - Material coefficients
Analysis1.step(3).soil.materialCoeffs.tanPhiCoeff = 1.1;
Analysis1.step(3).soil.materialCoeffs.shearStrengthCoeff = 1.2;
Analysis1.step(3).soil.materialCoeffs.skinFrictionCoeff = 1.3;
Analysis1.step(3).soil.materialCoeffs.pileTipResistanceCoeff = 1.4;
This command is scripted. Material coefficients can be seen as safety factors used to modify the soil
properties without changing the actual values. The actual values are divided by the coefficients meaning that
a coefficient greater then one (1) reduces the capacity. For more details, see User Manual Volume 2 or 6.
Nonlinear Calculation - Soil Curve generation
Analysis1.step(3).soil.curveGeneration.lowestLevelWithCyclicPY = 1 m;
Analysis1.step(3).soil.curveGeneration.lowestShearStiff = 120 KPa;
Analysis1.step(3).soil.curveGeneration.zoneOfInfluenceTZ = 20;
Analysis1.step(3).soil.curveGeneration.curveShapeFactorTZ = 0.85;
This command is scripted. For more details, see User Manual Volume 2 or 6.
Nonlinear Calculation - Group effects
Analysis1.step(3).soil.groupEffects. averagePoissonRatio = 0.6;
Analysis1.step(3).soil.groupEffects. modulusOfElasticity(10000 KPa,0 m, 20000 KPa,-150 m);
Group interaction effects (pile-soil-pile interaction) are calculated from the elastic half-space theory assuming
an E-modulus that increases linearly with depth. For more details, see User Manual Volume 2 or 6.
Nonlinear Calculation - Loads at surface
Analysis1.step(3).soil.loadsAtSurface.verticalStressAtSurface = 200 KPa;
Analysis1.step(3).soil.loadsAtSurface.verticalStressUnderEmbankment = 100 KPa;
Analysis1.step(3).soil.loadsAtSurface.widthOfEmbankmentSlopingPart = 1 m;
Analysis1.step(3).soil.loadsAtSurface.distancePileToEmbankmentToe = 0.5 m;
Analysis1.step(3).soil.loadsAtSurface.verticalStressUnderCircularLoadedArea = 50 KPa;
This command is scripted. Different types of vertical loading may exist at the soil surface in the vicinity of the pile.
Such loading will influence the vertical stress down along the pile and hence the lateral resistance.
For more details, see User Manual Volume 2 or 6.
Nonlinear Calculation - Splice
Analysis1.step(3).splice.solver.maxIterations = 25;
Analysis1.step(3).splice.solver.convergenceCriterion = 0.0015 m;
This command is scripted. The solver settings that are used to control the Splice analysis are Max iterations
and Convergence criterion. For more details, see User Manual Volume 2 or 6.
Linear Calculation
Analysis1 = Analysis(true);
Analysis1.add(MeshActivity());
Analysis1.add(WaveLoadActivity (Condition1));
Analysis1.add(PileSoilAnalysis (Condition1));
Analysis1.add(LoadResultsActivity());
Analysis1.step(3).nonlinearAnalysis(false);
This command is scripted. It is possible to neglect the pile and soil analysis by inserting fixed boundary conditions
along the piles and disregard the pile-soil analysis. This is the same as executing a wave and structural analysis where
all piles are fixed at the finite element positions. For more details, see User Manual Volume 2 or 6.
It is required to specify the pile boundary conditions to "Fixed" from the mesh activity.
Linear Calculation - Data check only
Analysis1.step(3).dataCheckOnly(true);
This command is scripted. The data check only option will run the Gensod activity,
but only check the input data for the Splice analysis. For more details, see User Manual Volume 2 or 6.
Linear Calculation - Automatic generation of input files
Analysis1.step(3).generateInput = false;
This command is scripted. The default option ("True") implies that the input files for Gensod and Splice
are automatically generated. In other words, if you modify one of the parameters this file is automatically updated
prior to the next analysis run. If you want to have a manual control of the input file you should deselect this option.
For more details, see User Manual Volume 2 or 6.
Running only Gensod step to enable Usfos export without running Splice
Analysis1 = Analysis(true);
Analysis1.add(MeshActivity());
Analysis1.add(WaveLoadActivity (Condition1));
Analysis1.add(PileSoilAnalysis (Condition1));
Analysis1.add(LoadResultsActivity());
Analysis1.step(3).gensodOnly();
This command is scripted provided that you tick off for "Journal activity executions".
When the user want to export to Usfos without running Splice, the 'gensodOnly()' command must be given.
This will prevent execution of Sestra Reduction, Splice and Sestra Retracking steps and also modify the GUI.
It is required to define a wave load condition that can be referred to when defining the wave load activity
(Insert|Environment|Deterministic Time Condition). For more details, see User Manual Volume 2 or 6.
